CPAC SB&M

Flow รวม Inventory

สรุปภาพรวมการทำงานของระบบ CPAC SB&M Inventory Management
ตั้งแต่ผู้ดูแลระบบเตรียมข้อมูล จนถึงพนักงานรับเข้า เบิกสินค้า และปิดงาน
สถานะใบเบิกในระบบ
pending: รอผู้จัดการอนุมัติ approved: อนุมัติแล้ว / รอจ่ายสินค้า completed: จ่ายสินค้าแล้ว cancelled: ยกเลิกและคืนยอดจอง
ฝั่ง Admin / Manager / พนักงานคลัง
01

Login SSO

เข้าสู่ระบบผ่าน RMC SSO และโหลดสิทธิ์ employee, manager หรือ admin

02

ตั้งค่า Master Data

Admin ดูแลข้อมูลสินค้า SKU หมวดหมู่ หน่วยนับ ราคา ต้นทุน สถานะใช้งาน และตำแหน่งจัดเก็บ

03

รับเข้าสินค้า

พนักงานคลัง/Manager/Admin เลือกสินค้าจาก master หรือสร้างใหม่ แล้วบันทึกรับเข้า พร้อม lot และวันหมดอายุ

04

ตรวจภาพรวมสต็อก

ดูยอดคงเหลือ รายการใกล้หมดสต็อก สินค้าใกล้หมดอายุ และประวัติรับเข้า/เบิกจ่าย

05

อนุมัติใบเบิก

Manager ตรวจผู้ขอ รายการสินค้า จำนวน และต้นทุนรวม ก่อนเปลี่ยนสถานะเป็น approved

06

จ่ายของ / ปิดงาน

Admin ยืนยันจ่ายของจริง ดูใบเบิกสินค้า และปิดงานเป็น completed เพื่อเก็บประวัติย้อนหลัง

ฝั่งพนักงาน / ผู้ขอเบิก
01

ค้นหาสินค้า

เปิดหน้าเบิกจ่าย ค้นหาจากชื่อ SKU หมวดหมู่ หรือกรองประเภทสินค้า

02

เลือกจำนวนเบิก

เลือกสินค้าและจำนวน ระบบตรวจคงเหลือไม่ให้เกินยอดที่มีอยู่

03

ระบบจัดล็อตอัตโนมัติ

เลือกล็อตจากวันหมดอายุใกล้สุดก่อน และสร้างรายการ out แบบ pending เพื่อจองสต็อก

LOT 1
EXP
LOT 2
FIFO
04

ส่งคำขออนุมัติ

กรอกผู้ขอ ผู้อนุมัติ หมายเหตุ และระบบส่งอีเมลแจ้งผู้จัดการ

05

ติดตามสถานะ

ดูใบเบิกของตนเอง รออนุมัติ ดูใบเบิกหลังอนุมัติ หรือยกเลิกก่อนปิดงาน

!

การแจ้งเตือน / ควบคุม

ระบบแจ้งเตือนสินค้าใกล้หมดที่ต่ำกว่า 5 หน่วย และสินค้าใกล้หมดอายุภายใน 90 วัน เพื่อให้คลังวางแผนเติมสต็อก

สรุปภาพรวม
Admin/Manager: SSO → Master Data → Dashboard → อนุมัติ/จ่ายของ → รายงาน | พนักงาน: รับเข้าสินค้า → ค้นหาสินค้า → เลือกจำนวน → ระบบจองล็อต → ส่งคำขอ → ติดตามสถานะ
ข้อมูลอ้างอิงจากหน้าจอจริงในระบบ: Overview, Items, Receive, Issue, Requisition Tracker, History, Master Data, Admin Rights และ Settings
